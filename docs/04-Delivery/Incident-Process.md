
# Incident Process

## Severity
- Sev1: outage / security breach
- Sev2: major degradation
- Sev3: minor issue

## Steps
1. Detect (alerts/support)
2. Triage (assign incident commander)
3. Mitigate (rollback/feature flag)
4. Communicate (status updates)
5. Resolve
6. Postmortem (root cause + actions)

## Postmortem template
- What happened?
- Impact
- Timeline
- Root cause
- Fix
- Preventive actions
